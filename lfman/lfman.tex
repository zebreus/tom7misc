\section{Implementation and acknowledgements}

\raggedleft \normalsize The LF Man game is implemented in ActionScript
with Flash VideoSystem.~~~~~~~The source code or codes is shown in
({\sc Figure~\ref{fig:sourcecode}}.) 

The authors are particularly grateful to the Internet People for their
help in learning ActionScript semantics. The best resource was
``steves tutes'', which can be found at
\verb+http://video-animation.com/flash_16.php+. Here 
we learned~~~about inheritance:

\begin{quotation}
When you use Inheritance you save time , because the base attributes and methods already exists,
And Reduce errors , because the base class , Shape has already been used and tested.
And You already know how the base class works. 
Using the same method name to implement different code in subclasses is called PolyMorphism. 

polymorphism does work. .. Try out all the other methods . Try and bust it.. bend it.. shake it..
\end{quotation}

We used inheritance in our VideoSystem application, where the Chicken
class is a subclass of the Man class, and the LFMan class is also a
subclass of the Man class, and the Man class has the implementation of
MovingInTheMazeOrMazes and other shared functionals.
Unfortunately:~~~we observed repeated degenerate behavior where the
Chicken behaves as an LF Man, mimicking his movements and warp-motion
to his locale. We tried using code from {\em steves tutes} to help us

\begin{quotation}
Here is a clas my good friend Lithium wrote as an example.
$\qquad\vdots$ \\
\verb+  var poop = _root.createEmptyMovieClip(name, level);+ \\
\verb+  poop.lineStyle(2, 0x000000, 100);+ \\
$\qquad\vdots$
\end{quotation}

Also unfortunately, this did not accord. In addition to introducing
compile error messages which we do not understand (but that do not
rampart our program from running), it created line-drawings on the
screen that did not appear desirable or related to the chicken
mimicking problem.

Instead we developed a new technique of inheritance which we call {\em
includeitance}. Instead of using the \verb+extends+ keyword to create
a subclass or subclasses, we use the \verb+#include+ keyword at the
head of the class so that LFMan and Chicken can have the same maze
motion but the Chicken does not think he is also a LFMan.
